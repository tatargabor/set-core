# Design Integration

How to ensure your orchestrated agents implement the correct visual design — not generic framework defaults.

## Why This Matters

Without explicit design context, agents use shadcn/Tailwind defaults: white backgrounds, system fonts, generic spacing. The Figma design exists but doesn't reach the agents automatically — you need to bridge the gap before running the sentinel.

**The rule**: Agents implement exactly what the spec tells them. Vague spec → generic result. Spec with exact tokens (`#78350F`, `Playfair Display`) → matches the design.

## Design Files

The pipeline uses three design files. All are optional — agents work without them, just with worse results.

| File | Purpose | Generated By |
|------|---------|-------------|
| `docs/design-system.md` | Design tokens (colors, fonts, spacing, radii) | `set-design-sync` or manual |
| `docs/design-brief.md` | Per-page visual descriptions (layout, components, responsive) | `set-design-sync` or manual |
| `openspec/changes/<name>/design.md` | Per-change design context (auto-generated) | Dispatcher (scope-matched) |

**How they flow:**

```
design-system.md  ─┐
                   ├──▶  Dispatcher  ──▶  per-change design.md  ──▶  Agent
design-brief.md   ─┘    (scope match)     (tokens + matched pages)
```

The dispatcher matches page names from `design-brief.md` against each change's scope, so agents only see the design sections relevant to their work.

## Supported Design Sources

| Source | Tool | Best For |
|--------|------|----------|
| Figma Make `.make` export | `set-design-sync` | Primary — richest data |
| Manual markdown | Write directly | When no Figma design exists |

## Step-by-Step Workflow

### 1. Design in Figma Make

Create your design in [Figma Make](https://www.figma.com/make). Include:
- A design token page (colors, typography, spacing, border-radius)
- Page layouts for each major view (homepage, catalog, detail, cart, checkout, admin)
- Component designs (header, footer, product card, buttons)

**Tip**: Give Figma Make a detailed prompt with exact hex colors, font names, and spacing values. These become the source of truth.

### 2. Export and Sync

```bash
cp ~/Downloads/my-design.make docs/design.make
set-design-sync --input docs/design.make --spec-dir docs/
```

This generates:
- **`docs/design-system.md`** — Design tokens only (colors, fonts, spacing)
- **`docs/design-brief.md`** — Per-page visual descriptions (layout, CTA text, component placement)

### 3. Review and Commit

```bash
# Check tokens
head -30 docs/design-system.md

# Check page descriptions
grep "^## Page:" docs/design-brief.md

# Commit both files
git add docs/design-system.md docs/design-brief.md
git commit -m "design: sync from Figma Make export"
```

### 4. Start the Sentinel

```bash
curl -X POST http://localhost:7400/api/<project>/sentinel/start \
  -H 'Content-Type: application/json' -d '{"spec":"docs/spec.md"}'
```

## Design Compliance (CRITICAL)

The review gate checks design compliance. **Mismatches are CRITICAL** — the agent must fix them or the change won't merge.

What's checked:
- Color values match design tokens (hex values, not CSS variables)
- Font families match design system
- Icon usage matches design patterns
- Layout patterns (grid/flex) match design structure

### When Design Compliance Fails

If a change fails the review gate with design CRITICAL findings:

1. **Check if the design files are detailed enough** — vague `design-brief.md` → agent can't follow → fail
2. **Update design files on main** — add missing detail to `design-brief.md` or `design-system.md`
3. **Commit to main** — `git add docs/design-*.md && git commit`
4. **The sentinel auto-redispatches** — the retry syncs main into the worktree, agent sees updated design

```
Agent fails design review
    ↓
You update docs/design-brief.md on main (commit)
    ↓
Sentinel redispatch → worktree syncs main → agent retries with new design
    ↓
Agent follows updated design → review passes
```

**Key insight**: Design files are living documents. If the agent can't implement a design spec, the spec needs more detail — not the gate needs weakening.

## What Each File Contains

### `design-system.md` — Tokens

```markdown
# Design System

## Design Tokens
### Colors
- `color-primary`: `#78350F`
- `color-background`: `#FFFBEB`

### Typography
- `font-heading`: `'Playfair Display', serif`
- `font-body`: `'Inter', sans-serif`

### Spacing
- `spacing-base`: `8px`
- `radius-card`: `6px`
```

### `design-brief.md` — Visual Descriptions

```markdown
# Design Brief

## Page: Home

HEADER (sticky):
- Left: Logo (Playfair Display, #78350F)
- Center nav: "Products" | "Stories" | "Subscribe" — Inter 16px
- Right: Search, Cart with badge, User avatar

HERO BANNER (full-width, ~500px):
- Coffee photo background
- Overlay: "Specialty coffee, delivered." — h1, white
- CTA: "Explore our coffees →" — filled #78350F, white text

## Page: ProductCatalog

GRID (4 columns):
- Product card: image (4:3), name, price, rating stars
...
```

## Updating the Design

When the Figma design changes:

1. Re-export the `.make` file
2. `set-design-sync --input docs/design.make --spec-dir docs/`
3. Review the diff — `git diff docs/design-*.md`
4. Commit: `git add docs/design-*.md && git commit -m "design: update from Figma"`
5. Running orchestration picks up changes on next redispatch/worktree sync

## Manual Design System

If you don't use Figma Make, create both files manually:
- `docs/design-system.md` — tokens (colors, fonts, spacing as key-value pairs)
- `docs/design-brief.md` — per-page layout descriptions (use `## Page: Name` headers)

The dispatcher reads them automatically — no `.make` file needed.

## Troubleshooting

| Problem | Cause | Fix |
|---------|-------|-----|
| Agent uses white background | Design tokens not in dispatch context | Verify `design-system.md` exists with hex values |
| Agent uses Inter instead of custom font | Font not in design tokens | Check `### Typography` in `design-system.md` |
| Only homepage matches design | Scope doesn't mention page names | Ensure spec mentions page names for keyword matching |
| Design review CRITICAL but agent can't fix | `design-brief.md` too vague | Add more detail to the page description, commit, let retry pick it up |
| Agent ignores design entirely | Per-change `design.md` not generated | Check dispatcher logs — is `design-brief.md` readable? |
