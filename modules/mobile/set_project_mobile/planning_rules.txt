## Mobile (Capacitor) Planning Rules

### Change Separation
- **Separate web changes from native changes.** A change that modifies only web code (src/, pages/, components/) should NOT touch ios/App/ or android/app/. Native changes are slower to verify (Xcode build) and create serialization bottlenecks.
- **Capacitor plugin additions are their own change.** Adding a new Capacitor plugin (e.g., @capacitor/camera) touches package.json, requires `cap sync`, and may need native config (Info.plist permissions). Keep it isolated.
- **Share Extension is foundational.** If the app needs a Share Extension, plan it as an early change — later changes may depend on the sharing pipeline. Share Extension requires: separate target, App Groups entitlements, shared UserDefaults, URL scheme for returning to main app.
- **Device identity is foundational.** If the app uses device-based identity (UUID in Capacitor Preferences), this must be one of the first changes — API auth, Share Extension auth, and user state all depend on it.
- **Deep linking changes touch both native and web.** URL scheme registration (Info.plist) + AppDelegate handling + web-side routing must be coordinated in a single change.

### Platform Testing
- **E2E tests run in the browser, not on device.** Capacitor apps are web apps — Playwright tests cover the web layer. Native-only features (push notifications, deep links) need manual verification or dedicated native test changes.
- **Build verification includes `cap sync`.** After web build, `npx cap sync` must succeed. The build gate runs both steps.
- **Static export limitations.** When `webDir` is `out/` (static export), API routes and middleware do NOT work inside the native app. E2E tests should account for this — test API routes via the web version, not the native shell.

### Native Code Constraints
- **Xcode project files are merge-hostile.** .xcodeproj/project.pbxproj is XML that conflicts easily. Never have two parallel changes modifying Xcode project structure.
- **Info.plist changes need review.** Permission strings, URL schemes, and App Transport Security exceptions are security-sensitive.
- **App Groups require matching entitlements.** If a change adds App Groups (for Share Extension), both the main app and extension must have matching group identifiers in their `.entitlements` files. Mismatch causes silent data loss.
- **Share Extension has a 25-second timeout.** iOS kills extensions that exceed this limit. Long operations (video processing, large file downloads) must be deferred to the main app via App Groups.
- **SPM, not CocoaPods.** Modern Capacitor uses Swift Package Manager. Do not create a Podfile unless a specific dependency requires it.

### Capacitor Config
- **capacitor.config.ts is cross-cutting.** Multiple plugins may need config entries. Treat it like layout.tsx — serialize changes that modify it.
- **webDir must match build output.** For Next.js static export: `out/`. For standard builds: `dist/` or `build/`. Mismatch causes blank screens on device.
- **AppDelegate is cross-cutting.** Device ID sync, URL scheme handling, Share Extension result injection, and push notification registration all live in AppDelegate. Serialize changes that modify it.

### Inter-Process Communication
- **App Groups UserDefaults is the primary IPC mechanism.** Share Extension writes data → Main app reads on resume. Always use a shared suite name (`group.<bundle-id>`).
- **Base64-encode data for JS injection.** When AppDelegate passes data to the WebView via `evaluateJavaScript`, encode as base64 to avoid JSON escaping issues.
- **Never store secrets in App Groups.** Use Keychain with a shared access group for sensitive data (auth tokens, API keys).
