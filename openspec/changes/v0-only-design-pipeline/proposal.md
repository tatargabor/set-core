# Proposal: v0.app Export as Production Design Source (v0-only)

## Why

The Figma Make pipeline (`.make` binary → `design-system.md` + `design-brief.md`) has consistently failed to produce reliable design output:

1. **Token mismatches** — the `.make` parser produced identical values for shadcn token pairs (`--muted` / `--muted-foreground`), causing invisible text in production runs.
2. **Missing interactive states** — Figma frames did not encode hover, selected, disabled states; agents had to invent them, producing unusable variant selectors.
3. **Opaque binary** — `.make` is a 3.9MB ZIP, undiffable, unreviewable, with duplicate sections going unnoticed for hundreds of lines.
4. **Agent re-implements from text** — agents read a markdown spec and reconstruct components from scratch. Even when the spec is good, every reconstruction introduces drift (wrong className, swapped shadcn primitive, restructured layout).

v0.app (Vercel) generates Next.js + shadcn/ui + Tailwind code natively. Its output is exactly the stack we use. By treating the v0 export as the canonical design source — not a markdown intermediate — agents become **integrators**, not **re-implementers**: they copy v0's TSX, then adapt the data layer, copy, and types.

This change archives the entire Figma pipeline and switches set-core to v0-only design source.

## What Changes

### Source pipeline

- **NEW**: `set-design-import` CLI (web module) imports a v0.app design into a scaffold from one of two sources:
  - **Primary: external git repo** (`--git <url> --ref <sha-or-tag>`) — works with any provider (GitHub/GitLab/Bitbucket/self-hosted), public or private, via standard git auth (SSH keys, credential helpers, env vars). v0.app's "Push to GitHub" feature is the natural producer.
  - **Fallback: ZIP export** (`--source <zip>`) — for offline use cases or when git is unavailable.
- **NEW**: `scaffold.yaml` declares the design source (URL + ref OR zip path). Re-import = `git fetch` + checkout new ref (or unzip new file). No version drift between team members.
- **NEW**: `v0-export/` directory is **gitignored in scaffolds** — materialized at deploy time from the git repo or ZIP. Avoids set-core repo bloat (~1-5MB per scaffold) and keeps design history in the v0 repo's own commit log (diffable across iterations).
- **NEW**: `design-manifest.yaml` per scaffold — maps routes and shared components to scope keywords for dispatcher matching. Auto-generated by `set-design-import`.
- **NEW**: design-source slice — when dispatching a change, the dispatcher copies the scope-matched v0 files into `openspec/changes/<name>/design-source/`. The agent reads these as starting point.

### Verification

- **NEW**: design-fidelity gate (web module) — standalone integration gate. Builds agent worktree + v0 reference with shared HU fixtures, screenshots each route at desktop/tablet/mobile, runs pixelmatch diff. Threshold-exceeding deviations block merge.
- **NEW**: build-time content substitution layer — renders v0-export with the project's HU seed data so screenshot diffs aren't polluted by copy differences.

### Removals (BREAKING)

- **BREAKING**: Figma `.make` parser (`lib/set_orch/design_parser.py` MakeParser) — deleted. Projects with `.make` files must regenerate their design in v0.
- **BREAKING**: Figma MCP fetcher (`lib/design/fetcher.py`) — deleted. No more Figma URL → snapshot pipeline.
- **BREAKING**: `set-design-sync` CLI — replaced by `set-design-import` (v0-only). Old name removed.
- **BREAKING**: `design-snapshot.md` / `design-system.md` artifacts — no longer generated by the framework. Tokens come from `globals.css` (synced from v0-export). Optional vibe notes can stay in `design-brief.md` but it is no longer the agent's design source.
- **BREAKING**: `figma_url` and related fields in `orchestration.yaml` — removed. Migration: delete the field; framework will use `v0-export/` automatically.

### Architectural placement

- **Layer 1 (`lib/set_orch/`, abstract)**: profile ABC adds `detect_design_source() → "v0" | "none"`. Dispatcher copies the design-source slice generically based on what the profile reports. NO v0-specific code in core.
- **Layer 2 (`modules/web/`)**: ALL v0 logic lives here — importer CLI, manifest parser, scope matcher, fixture-aware renderer, fidelity gate. WebProjectType implements `detect_design_source` (returns "v0" if `v0-export/` exists).

### Agent workflow change

- `templates/core/rules/design-bridge.md` rewritten for v0-only model: when `design-source/` is present, agent copies the TSX as the visual truth. Refactor allowed in code structure (extract components, rename, retype, wire backend); design integrity (classNames, JSX structure, shadcn primitive choice, spacing, animations) MUST remain untouched. The contract is verified by the fidelity gate.

### Migration path for existing consumers

Consumer projects currently using Figma must regenerate designs in v0 using the prompts in `docs/v0-prompts.md` (already added in the existing `v0-design-pipeline` change), then run `set-design-import --source <zip>` against their scaffold. No backward-compatibility shim — the old pipeline is deleted in this change.

## Capabilities

### New Capabilities

- **v0-export-import** — web module CLI for importing v0.app designs (git or ZIP) into scaffolds, validating structure + quality, generating manifest. Includes v0 template quality validation (tsc, build smoke, navigation integrity, naming consistency, variant coverage) → reports findings to `<scaffold>/docs/v0-import-report.md`.
- **v0-design-source** — web module dispatch context that copies scope-matched v0 files into `design-source/` per change.
- **design-fidelity-gate** — web module integration gate combining (a) AST-level structural skeleton check (route inventory, shared layout files, component decomposition) — runs FIRST, fails fast on cheap mismatches — and (b) Playwright visual regression with shared HU fixtures.
- **v0-fixture-renderer** — web module build-time substitution layer that renders v0-export with HU seed data for screenshot comparison.
- **decompose-design-binding** — orchestration plan integration: decompose skill reads `design-manifest.yaml`, generates `design_routes: [...]` per change, validates manifest coverage (every route assigned or explicitly deferred). Dispatcher prefers explicit `design_routes` over scope-keyword fuzzy matching.

### Modified Capabilities

- **design-bridge** — rule rewritten for v0-only model. Agent reads `design-source/` files as design truth; refactor policy explicitly defined.
- **design-dispatch-context** — replace Figma snapshot extraction with v0 source slice. Tokens read from `globals.css` (synced from v0-export).
- **design-dispatch-injection** — input.md gets `design-source/` pointer instead of `design.md` markdown spec.
- **design-verify-gate** — replace token-grep code review section with standalone visual regression gate (the new design-fidelity-gate capability).
- **profile-hooks** — add `detect_design_source()` ABC method to ProjectType.
- **profile-loader-builtin** — WebProjectType implements `detect_design_source` returning "v0" when `v0-export/` exists.

### Removed Capabilities

- **design-make-parser** — Figma `.make` parsing removed entirely. **Reason**: Figma pipeline never produced reliable output (token mismatches, missing states, opaque binary). **Migration**: regenerate design in v0 per `docs/v0-prompts.md`, run `set-design-import`.
- **design-snapshot** — Figma MCP snapshot generation removed. **Reason**: superseded by v0-export which contains the actual code, not a markdown approximation. **Migration**: same as above.
- **design-spec-sync** — Figma URL → design-system.md sync removed. **Reason**: no Figma input anymore. **Migration**: same as above.
- **design-brief-parser** — markdown brief parsing for dispatch context removed (brief becomes optional, non-authoritative). **Reason**: v0-export TSX is now the source; brief is a vibe note at most. **Migration**: scope-matching now uses `design-manifest.yaml` instead of brief sections.
- **design-brief-stem-match** — stem-based scope matching against brief sections removed. **Reason**: replaced by manifest-driven keyword matching. **Migration**: define scope_keywords in design-manifest.yaml.

### Relationship to in-progress `v0-design-pipeline` change

The existing `v0-design-pipeline` change (proposal + tasks + v0-prompts.md, 0/33 tasks done) covered Phase 1 (humans use v0 for design authoring, output still a markdown brief). This change supersedes it: v0 export is the source, not the brief.

The in-progress change SHOULD be archived as superseded once this change's proposal is approved. Its only durable artifact (`v0-prompts.md`) is moved to this change's `docs/` directory.
