# Spec: Design Make Parser (delta)

## REMOVED Requirements

### Requirement: REQ-MAKE-PARSE — Parse .make files into structured design-system.md

**Reason:** The Figma `.make` ingestion pipeline is removed entirely. The parser produced unreliable output (token mismatches, missing interactive states, opaque binary diffs hiding duplicates). Replaced by v0.app ZIP export consumed via `v0-export-import` capability.

**Migration:** Scaffold authors with existing `.make` files SHALL regenerate the design in v0.app (using prompts in `docs/v0-prompts.md`) and run `set-design-import --source <v0-export.zip>` against the scaffold. The `.make` file SHOULD be deleted from the scaffold repo.

### Requirement: REQ-MAKE-FORMAT — design-system.md output format

**Reason:** `design-system.md` is no longer generated by the framework. Design tokens come from `globals.css` synced from v0-export.

**Migration:** Token quick-reference for agents is now extracted at dispatch time from `shadcn/globals.css` by the web module's design-source provider. Existing scaffolds with `design-system.md` may delete the file.

### Requirement: REQ-MAKE-EXTENSIBLE — Support future input formats

**Reason:** The format-detection layer (`.make` / `.md` passthrough / unknown error) is removed because the only supported input format going forward is the v0 ZIP export, handled by `set-design-import`.

**Migration:** No equivalent mechanism is needed — `set-design-import` validates v0 ZIP structure directly and rejects mismatched inputs with a clear error.

### Requirement: REQ-MAKE-CLI — CLI interface

**Reason:** The `set-design-sync` CLI is removed (binary, source) along with its options (`--input`, `--spec-dir`, `--output`, `--dry-run`, `--format`). Replaced by `set-design-import` (web module) with v0-export-specific flags.

**Migration:** Update any scripts or runners that invoke `set-design-sync` to use `set-design-import --source <zip> --scaffold <dir>` instead. The runner `tests/e2e/runners/run-craftbrew.sh` is updated as part of this change. External consumers must update their deployment scripts.
