# Design: visual-e2e-smoke-tests

## 1. Console Error Capture in E2E Gate

The E2E gate already runs Playwright via `e2e_command`. The fix is to wrap the command with error detection — but since we don't control the user's test code, we need a different approach.

### Approach: Post-E2E Error Check

After the E2E command runs (pass or fail), inspect the test output for error indicators:

```python
# In _execute_e2e_gate(), after e2e_cmd_result:
error_indicators = [
    "Runtime Error",
    "Hydration failed",
    "Functions are not valid as a child",
    "Error: Minified React error",
    "pageerror",
    "console.error",
]
has_runtime_errors = any(ind.lower() in e2e_output.lower() for ind in error_indicators)
```

This catches errors that Playwright logs to stdout/stderr even when tests "pass" (e.g., page loads with HTTP 200 but has console errors).

### Approach: Smoke E2E Template (project-type generated)

The real fix is a **route smoke test** that the project-type plugin generates:

```typescript
// e2e/smoke-routes.spec.ts (auto-generated by set-project-web)
import { test, expect } from '@playwright/test';

const routes = [
  '/hu/kavek',
  '/hu/kavek/some-product-id',
  '/hu/eszkozok',
  // ... auto-discovered from app/ directory
];

for (const route of routes) {
  test(`${route} loads without errors`, async ({ page }) => {
    const errors: string[] = [];
    page.on('pageerror', err => errors.push(err.message));

    await page.goto(route);
    await page.waitForLoadState('networkidle');

    // No runtime errors
    expect(errors).toHaveLength(0);

    // No React error boundary
    const errorBoundary = page.locator('[data-nextjs-error]');
    await expect(errorBoundary).toHaveCount(0);

    // Page has content (not blank)
    const body = await page.locator('body').textContent();
    expect(body?.trim().length).toBeGreaterThan(0);
  });
}
```

### Profile Extension Point

```python
# NullProfile (set-core)
def generate_smoke_e2e(self, project_path: str) -> str | None:
    """Generate route smoke test content. Returns test file content or None."""
    return None

# WebProjectType (set-project-web) — implements this
def generate_smoke_e2e(self, project_path: str) -> str | None:
    routes = self._discover_routes(project_path)  # scan app/[locale]/**/page.tsx
    return self._render_smoke_template(routes)
```

## 2. Screenshot Configuration

Add `screenshot: 'on'` to the Playwright env when running E2E gate:

```python
e2e_env["PLAYWRIGHT_SCREENSHOT"] = "on"
```

Projects can configure this in `playwright.config.ts` via `process.env.PLAYWRIGHT_SCREENSHOT`.
